# 网站自动更新方案

## 目标

以后每次提供新文章素材后，尽量减少人工步骤，让网站自动更新文章页、首页文章入口、`sitemap.xml`、`llms.txt` 和上线包。

## 当前可立即执行的半自动流程

你提供素材后，我可以一次性生成：

- 公众号终稿；
- 小红书手动发布包；
- `AI摘要与元信息.md`；
- 知乎问答版；
- 网站文章版；
- 网站文章 HTML；
- 首页文章入口；
- `sitemap.xml`；
- `llms.txt`；
- Netlify 上传包。

你只需要把我生成的更新包拖到 Netlify 的 Production deploys 上传区。

## 建议升级的一键自动流程

### 方案一：GitHub + Netlify 自动部署

这是最稳的长期方案。

流程：

1. 建一个私有 GitHub 仓库，例如 `weibinlawyer-site`。
2. 把 `卫斌律师个人网站` 文件夹放入仓库。
3. Netlify 连接这个仓库。
4. 以后我更新网站文件后，只要提交到 GitHub，Netlify 会自动部署。

优点：

- 不需要每次拖拽上传；
- 有历史版本；
- 可以回滚；
- 更适合长期维护。

需要你提供或授权：

- GitHub 仓库；
- Netlify 连接 GitHub；
- 后续允许我提交网站更新。

### 方案二：Netlify Token 自动部署

流程：

1. 在 Netlify 生成 Personal Access Token。
2. 获取 Site ID。
3. 本地保存为环境变量或 `.env`。
4. 我可以通过命令直接部署更新。

优点：

- 不依赖 GitHub；
- 可以从本地直接发布。

注意：

- Token 属于敏感信息；
- 应只保存在本机，不应发到聊天里；
- 需要妥善管理权限。

### 方案三：继续手动拖拽

流程最简单，但每次需要人工拖拽。

适合当前过渡阶段。

## 推荐选择

建议用方案一：GitHub + Netlify 自动部署。

这样以后流程可以变成：

1. 你给素材；
2. 我生成公众号、小红书、知乎、网站文章；
3. 我更新网站文件；
4. 提交到 GitHub；
5. Netlify 自动上线；
6. 我给你上线链接和发布包。

## 每篇文章进入网站时的标准字段

- 标题；
- URL slug；
- 一句话摘要；
- 发布日期；
- 关键词；
- 文章 HTML；
- 作者简介；
- 免责声明；
- `Article` 结构化数据；
- `llms.txt` 条目；
- `sitemap.xml` 条目。
